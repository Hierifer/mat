# 麦克风权限设置指南

## macOS 系统设置

如果看到 "not allowed" 或 "麦克风权限被拒绝" 错误，请按照以下步骤操作：

### 方法 1: 系统设置（推荐）

1. **打开系统设置**
   - 点击屏幕左上角的  菜单
   - 选择 "系统设置..." (System Settings)

2. **进入隐私与安全性**
   - 点击左侧菜单中的 "隐私与安全性" (Privacy & Security)
   - 向下滚动找到 "麦克风" (Microphone)

3. **授权 Mat Terminal**
   - 点击 "麦克风"
   - 在应用列表中找到 "mat" 或 "Mat Terminal"
   - 勾选其左侧的复选框以授权

4. **重启应用**
   - 关闭 Mat Terminal
   - 重新启动应用
   - 再次尝试语音输入功能

### 方法 2: 首次提示授权

当你第一次点击麦克风按钮或使用快捷键时：

1. **系统会弹出权限请求对话框**
   ```
   "mat" would like to access the microphone.
   [Don't Allow]  [OK]
   ```

2. **点击 "OK" 允许访问**

3. **如果误点了 "Don't Allow"**
   - 请按照"方法 1"手动授权

### macOS 版本差异

**macOS Ventura (13.0) 及以上**:
- 系统设置 → 隐私与安全性 → 麦克风

**macOS Monterey (12.0) 及以下**:
- 系统偏好设置 → 安全性与隐私 → 隐私 → 麦克风

## Windows 系统设置

如果在 Windows 上遇到权限问题：

### Windows 11

1. **打开设置**
   - Win + I 打开设置
   - 选择 "隐私和安全性"

2. **麦克风权限**
   - 点击 "麦克风"
   - 确保 "麦克风访问权限" 已开启
   - 确保 "允许应用访问麦克风" 已开启

3. **浏览器权限**
   - 如果使用浏览器版本，还需要在浏览器中允许麦克风访问

### Windows 10

1. **打开设置**
   - Win + I 打开设置
   - 选择 "隐私"

2. **麦克风**
   - 左侧菜单选择 "麦克风"
   - 确保 "允许应用访问麦克风" 已开启

## Linux 系统设置

### Ubuntu/Debian

1. **检查麦克风是否工作**
   ```bash
   arecord -l
   ```

2. **测试录音**
   ```bash
   arecord -d 5 test.wav
   aplay test.wav
   ```

3. **权限问题**
   - 确保用户在 `audio` 组中
   ```bash
   sudo usermod -a -G audio $USER
   ```

4. **PulseAudio 设置**
   ```bash
   pavucontrol
   ```

## 浏览器权限

如果使用 Web 版本或开发模式：

### Chrome/Edge

1. 点击地址栏左侧的锁图标
2. 找到 "麦克风" 设置
3. 选择 "允许"

### Safari

1. Safari → 设置 → 网站
2. 选择 "麦克风"
3. 找到对应网站并选择 "允许"

### Firefox

1. 点击地址栏左侧的 🔒 图标
2. 点击 "连接安全" 右侧的 >
3. 点击 "更多信息"
4. 选择 "权限" 标签
5. 找到 "使用麦克风" 并取消 "使用默认设置"
6. 选择 "允许"

## 故障排除

### 权限已授予但仍然不工作

1. **重启应用**
   - 完全退出 Mat Terminal
   - 重新启动

2. **检查麦克风硬件**
   - 确保麦克风已连接
   - 在其他应用中测试麦克风

3. **检查系统音量**
   - 确保麦克风输入音量不为 0
   - 确保麦克风未静音

4. **查看控制台日志**
   - 打开开发者工具 (F12)
   - 查看 Console 中的错误信息
   - 查找 `[Speech]` 标签的日志

### 仍然无法解决

如果以上方法都无法解决问题，请：

1. **收集信息**
   - 操作系统版本
   - 浏览器类型和版本（如果适用）
   - 错误信息截图
   - 控制台日志

2. **提交 Issue**
   - 访问: https://github.com/Hierifer/mat/issues
   - 创建新 Issue 并附上收集的信息

## 隐私说明

- Mat Terminal 的语音识别功能使用浏览器内置的 Web Speech API
- 语音数据由浏览器处理，可能会发送到浏览器厂商的服务器（如 Google）
- Mat Terminal 本身不会存储或上传你的语音数据
- 转录的文字仅用于输入到终端，不会被保存或传输

---

**重要提示**：授权麦克风权限后，务必重启 Mat Terminal 应用以使设置生效。
